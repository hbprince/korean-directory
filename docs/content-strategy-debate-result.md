## 1. 현재 상황 진단 — 4팀 공통 인식

### 핵심 문제
- **트래픽 52명/주** → 어떤 기능을 만들어도 사용자가 없으면 무의미
- **"전화번호 목록"** → 구글/Yelp/카카오톡과 차별화 없음
- **데이터 신뢰도 불확실** → 62,570개 중 실제 영업 중 추정 55~65% (약 34,000~40,000개)
- **editorialSummary 0%** → Google Places 데이터 활용 미비

### 핵심 기회
- 경쟁사(라디오코리아, 한국일보, 헤이코리안) **AI/LLM 미활용**
- 한국어 long-tail 키워드 **경쟁 극히 낮음**
- 비밀집 지역(교외, 중소도시) 한인들에게 **디렉토리가 유일한 정보원**
- 신규 이민자의 정보 갈증 → **가이드 콘텐츠가 킬러**

## 2. 전략별 토론 결과

### 전략 A: Google Places 데이터 보강

**4팀 합의: "기반 인프라. 반드시 먼저 해야 하지만 이것만으로는 부족"**

| 항목 | 수치 |
|------|------|
| 개발 공수 | 4.5일 |
| API 비용 | $1,960 (40,000건 기준) |
| 월 운영비 | $100~300 |
| 생성 콘텐츠 | 신규 0, 기존 62,570 페이지 개선 |
| SEO 영향 | 즉시 (기존 페이지 품질 향상) |
| 예상 트래픽 증가 | +10~25명/주 |

**Product/UX팀 핵심 주장:**
> "LLM 리뷰 요약(한국어)은 한인맵의 핵심 차별화 요소. 구글/Yelp 영어 리뷰를 한국어 3줄로 요약하면 '이것 때문에 다시 온다' 수준의 기능이 된다."

**기술팀 핵심 분석:**
> "기존 인프라 100% 활용 가능. fields 파라미터에 editorial_summary 추가 + upsert 로직 10줄 변경으로 즉시 실행. Google ToS attribution만 준수하면 리스크 낮음."

**커뮤니티전문가 핵심 주장:**
> "데이터 품질이 신뢰를 결정한다. 폐업 업소에 전화하는 경험 한 번이면 재방문 없다. 영업 여부 검증이 1순위."

**쟁점: editorialSummary에 대한 기대치**
- 기술팀 경고: Google Places API가 한인 소규모 업소에 editorialSummary를 제공하지 않을 가능성 높음
- 대안 합의: **LLM이 리뷰 + types + 기본 정보로 자체 요약 생성** (Google 데이터에만 의존하지 않기)

### 전략 B: LLM 생성 가이드 콘텐츠

**4팀 합의: "트래픽 확보의 핵심 엔진. 52명→500명의 70%를 담당"**

| 항목 | 수치 |
|------|------|
| 개발 공수 | 8일 |
| LLM API 비용 | $89 (1,750개 가이드, Claude Sonnet) |
| 월 운영비 | $5~10 (월간 업데이트) |
| 생성 콘텐츠 | 1,750개 신규 페이지 |
| SEO 영향 | 1~3개월 후 |
| 예상 트래픽 증가 | +80~200명/주 (8~12주 후) |

**SEO팀 핵심 주장:**
> "가이드 콘텐츠가 트래픽의 60~70%를 만든다. 한국어 long-tail 키워드는 경쟁이 극도로 낮아 50개 가이드만으로 주 120명 달성 가능. 단, 대량 생성이 아닌 점진적 출판(주 15~25페이지)이 필수."

**Product/UX팀 킬러 콘텐츠 5선:**
1. "미국 신규 이민자 정착 가이드: 도착 후 30일 체크리스트" → 시나리오 5 (정착)
2. "미국 이민 변호사 선택 완벽 가이드" → 시나리오 2 (변호사 비교)
3. "미국 세금 신고 가이드: CPA 선택법" → 시나리오 4 (시즌성)
4. "LA 한인 병원 찾기: 보험별 추천 가이드" → 시나리오 1 (긴급 의료)
5. "미국 첫 집 구매 가이드: 모기지부터 클로징까지" → 시나리오 6 (부동산)

**커뮤니티전문가 핵심 주장:**
> "한인 이민자가 가장 절실하게 필요로 하는 것은 '미국 시스템을 한국어로 설명해주는 가이드'. 라디오코리아, 한국일보 등은 뉴스에 집중하여 이 니즈가 비어있다."

**쟁점: LLM 콘텐츠의 Google 패널티 리스크**

| 위험 | 완화 전략 |
|------|----------|
| 대량 생성 감지 | 점진적 출판 (Phase 1: 50개 → Phase 2: 150개 → Phase 3: 300개) |
| 템플릿 반복 | haninmap 고유 데이터 삽입 ("등록 업소 47곳 중 평점 4.5 이상 12곳") |
| thin content | 가이드 당 1,500자+ + FAQ 5~10개 + 관련 업소 연결 |
| 신뢰도 | AI 초안 + 전문가 감수 형태로 발행 |

### 전략 C: 한인 커뮤니티 이벤트/공지 자동 수집

**4팀 합의: "SEO freshness 신호에 가치 있으나, 단독으로는 약하다. Phase 2에서 보조 전략으로"**

| 항목 | 수치 |
|------|------|
| 개발 공수 | 11.5~13.5일 |
| 비용 | $0 (크롤링) |
| 월 운영비 | $0~20 |
| 생성 콘텐츠 | 월 50~200개 이벤트 페이지 |
| 예상 트래픽 증가 | +30~80명/주 (직접 + freshness 간접 효과) |

**SEO팀 숨겨진 가치 발견:**
> "이벤트 콘텐츠 자체의 트래픽은 제한적이지만, 사이트 전체의 크롤링 빈도와 freshness 신호를 높여 다른 페이지의 순위에 간접 효과가 크다. Google은 자주 업데이트되는 사이트를 더 자주 크롤링한다."

**기술팀 경고:**
> "파서 유지보수 부담이 지속적으로 발생. 한인회/총영사관 사이트 구조 변경 시 파서 재작성 필요. 카카오톡, Facebook은 크롤링 불가."

**커뮤니티전문가 반론:**
> "한인 이벤트 정보는 이미 카카오톡과 Facebook에서 충분히 유통된다. 같은 노력이면 가이드 콘텐츠에 투자하는 것이 ROI가 압도적으로 높다."

**합의:** 총영사관 공지만 선별적으로 시작 (구조 안정적, 공식 정보)

### 전략 D: 사용자 참여형 콘텐츠

**4팀 합의: "장기적 가치 높지만 현 시점에서 시기상조. 주 500명+ 이후 도입"**

| 항목 | 수치 |
|------|------|
| 개발 공수 | 29일 |
| 비용 | 인증시스템 구축 필요 |
| 월 운영비 | $20~50 (이메일 + 모더레이션) |
| 생성 콘텐츠 | 사용자 의존 (불확실) |
| 예상 트래픽 증가 | +5~15명/주 (6개월 후, 불확실) |

**Product/UX팀 핵심 반론:**
> "평균 점수가 가장 높지만 '닭과 달걀' 문제가 치명적. 52명/주 트래픽에서 후기는 안 쌓이고, 빈 Q&A 섹션은 사이트 신뢰도를 파괴한다."

**기술팀 핵심 반론:**
> "현재 인증 시스템이 전혀 없음. 29일 개발은 프로젝트 아키텍처의 근본적 변화를 의미. 읽기 전용 → 쓰기 가능은 CSRF, 스팸, 모더레이션 등 전혀 다른 복잡도."

**커뮤니티전문가 핵심 반론:**
> "한인들은 Yelp도 잘 안 쓴다. 리뷰 작성률이 미국 평균의 1/3~1/5. 실명 리뷰에 대한 문화적 거부감(커뮤니티가 좁아 신원 노출 우려), 업주와의 관계 의식. UGC는 한인 문화와 가장 맞지 않는 전략이다."

**합의:** 업주 클레임만 트래픽 500명/주 달성 후 도입. Q&A는 1,000명/주 이전에는 절대 도입하지 않을 것.
